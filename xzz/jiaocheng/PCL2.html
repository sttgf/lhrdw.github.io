核心功能概述

PCL2是功能强大的Minecraft启动器，支持Forge与Fabric双模组加载器，提供版本隔离、内存分配、模组管理和联机功能。本指南将帮助您快速掌握核心操作流程和两种加载器的选择策略。

PCL2基础使用教程

1 版本管理

创建版本

打开PCL2 → 点击左侧「下载」 → 选择游戏版本(如1.20.1) → 勾选 Forge/Fabric → 点击「开始安装」

提示：安装前在「版本设置」中开启 版本隔离，避免模组冲突

切换版本

左侧版本列表 → 右键目标版本 → 点击「设为当前版本」 → 启动游戏

2 游戏启动配置

内存分配

版本设置 → Java设置 → 调整最大内存(建议4GB以上)

启动参数

可自定义窗口标题、分辨率(如 --width 1280 --height 720)

3 模组/资源包管理

添加模组

右键版本 → 打开「模组文件夹」 → 将 .jar 文件拖入(无中文路径！)

资源包/光影

拖入 .zip 文件至对应文件夹(resourcepacks 或 shaderpacks)

4 联机方法

局域网联机

主机开局域网 → 记录端口号 → PCL2点击「联机」生成连接码 → 好友输入连接码加入

服务器联机

直接输入服务器IP(需服务端与客户端模组一致)

Forge与Fabric的区别与使用

对比项 Forge Fabric
适用场景 经典大型模组(1.12.2/1.7.10) 高版本优化/轻量模组(1.18+)
性能 资源占用高，兼容复杂 轻量化，帧率优化明显(如钠模组)
光影支持 需OptiFine(易冲突) 钠(Sodium)+Iris组合(兼容性更好)
联机要求 服务器需Forge服务端 服务器需Fabric Loader

Forge操作要点

  • 安装后需确认依赖库(自动修复提示)
  • 模组文件直接拖入 mods 文件夹(无需额外API)

Fabric操作要点

  • 必须安装 Fabric API(否则模组不生效)
  • 推荐模组组合：钠(Sodium)+锂(Lithium)+磷(Phosphor)+Iris(性能与光影兼容最佳)

如何选择Forge还是Fabric？

选Forge

玩经典模组整合包(如"虚无世界""工业时代")

选Fabric

高版本生存优化(如1.20.1+光影流畅运行)

常见问题解决

游戏崩溃

检查模组与游戏版本、加载器是否匹配

删除冲突模组(从日志文件 logs/latest.log 定位错误)

联机失败

主机与客户端版本、加载器、模组列表必须完全一致！

关闭防火墙或检查端口转发(联机需开放TCP/UDP端口)

光影加载异常

Forge：更新OptiFine至最新版

Fabric：检查钠和Iris版本兼容性(如Iris 1.6.x仅支持1.20.1)

操作流程图

启动PCL2 → 创建版本(Forge/Fabric)

安装模组 → 配置Java/内存

启动游戏

联机：开局域网 → 分享连接码 → 好友输入加入

崩溃：检查日志 → 删除冲突模组 → 重试
